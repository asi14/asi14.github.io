Data Conversion

Using primative types inevitably requires conversions between them- there are two kinds of conversions:

  • Widening Conversions: Conversions that convert into a primative type of same or higher complexity. Example: int to double (integer -> decimal).
    Note: These are the best conversions because you won't lose any information in the process.
  • Narrowing Conversions: Conversions that convert into a primative type of lower complexity. Example: double to int (decimal -> integer).
    Careful! You'll lose information about your values if you use this type of conversion!
  • boolean doesn't apply here because it can't be converted to another primative type.

Examples

public class Conversions{
  public static void assignConvert(){
    double money;
    int dollars = 5;
    money = dollars; //dollar's int value gets turned into money's double value
                     //also an example of Widening Conversions
  }
  public static void arithmeticPromotion(){
    double sum = 5.6;
    int count = 3;
    double reuslt = sum / count; //count's int value is converted into double (float) since sum and reuslt is double
                                 //Note how sum is a float, so the output is a float either way
  }
  public static void casting(){
    int dollars;
    double money = 5.6;
    dollars = (int) money; //the desired type of conversion is put next to the var being converted
                           //also an example of Narrowing Conversions, as the decimal part of money gets chopped off
                           //has useful applications in rounding
  }
}

Anatomy of a Class

//syntax: [tag] class [ClassName]
public class BookTest{ //this is the MAIN class because it has the "main function"
  //method syntax: [public/private/protected/none][void/type] methodName([type] parameter name)
  public static void main(String[] args){ //this is the syntax for the MAIN method. (best memorize it)
    Book book1 = new Book(3); //the 'new' operator declares a new object, instantiated from a class.
    //object declaration syntax: [class] varname = new [class](parameters);
    for(int i = 1; i<=3; i++){
      book1.nextPage(); //this calls a method
      System.out.println(book1.getCurrentPages()); //so does this
    }
  }
}

class Book{ //this is a second class in the program.
  //these are FIELDS, or INSTANCE VARIABLES (vocab is the same)
  //note how these are PRIVATE because you're ENCAPSULATING these inside the class.
  private int numPages;
  private int currentPages;
  public Book(int n){//this is called a CONSTRUCTOR, because this sets up the object. more often than not it defines fields.
    //these lines are giving values to the fields, for other classes to use.
    numPages = n;
    currentPages = 1;
  }

  //these next two methods are called ACCESSOR methods, as you're ACCESSING a field and returning it outside a class.
  public int getNumPages(){
    return numPages;
  }
  public int getCurrentPages(){
    return currentPages;
  }


  public void nextPage(){
    if(currentPages < numPages){
      currentPages+=1;
    }
  }
}

Instance Data

Where a variable is declared determines its scope-- where it can be used. Here, take a look.

public class scopeEx{
  //these fields have a scope that spans the ENTIRE CLASS, because they're declared outside all methods.
  //that's the nifty thing about fields, actually because you can toss them around basicaly.
  private int field1;
  private String field2;
  public void method1{
    int variable1 = 4; //this variable has a scope of the entire method, and can be used by all program/control flows.
    while(variable1 != 10){
      variable1+=1;
      int count = variable1; //this variable has a scope of only inside the while loop, and can't be used outside.
    }
    System.out.println(count); //this gives an error because count is in the scope of the while loop.
  }
}

Actually, scopeEx illustrates what encapsulation is pretty well. It hides its internal variables (the fields field1 and field2 from other classes), and does so using the modifier private. There's two other modifiers: public and final (variable value can't be changed any further).

Anatomy of a Method

I'm gonna bring back that Book program from earlier to illustrate how Method Declaration and Program Flow work.

public class BookTest{

  public static void main(String[] args){ //this is the syntax for the MAIN method. (best memorize it)

  }

Also important is return- it serves two roles in Java. If a method's return type is set to a partiular type (see int in getCurrentPages()), return allows you to return a variable/value of that type. (If you tried returning a string you're getting an error) But if the return type is set to void (see nextPage()), then return exits the program flow out of the method into the next higher-up method.(return has no val/variable next to it cause it's void, otherwise an error gets triggered)

For example, if nextPage() had a return keyword, it would send the control flow up towards the main(String[] args) method since it called nextPage().

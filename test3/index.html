  //you can also think of them as "placeholders" to be plugged in when the object is created.
  private String name;
  private int age;

  //String n and int a these are examples of PAREMETERS, and are placeholders for the programmer to plug in
  public Student (String n, int a)
  {
    name = n + " Cat:";
    age = a + 100;
  }

  public void showData()
  {

    //this is an example of a LOCAL VARIABLE: it exists within the method showData() only
    //once the program is done with showData(), then oldAge will be deleted
    int oldAge = 1000;

    for (int k = 1; k < 5; k++)
    {
      oldAge = oldAge + age;
    }
     System.out.print(name + " " + age + "!  ");
  }
}

Casting is a way to forcibly convert data types into other data types. It's the easiest way to do so, but it' easier to not convert unless needed.

Careful! Narrowing and Widening Conversion rules apply
public class CastDemo{
  public static main(String[] args){
    double thingy = 123;
    int num1 = 3;
    double num2 = (double) num1; //converts num1 into a double
    int result = (int)(thingy / num2); //outputs int because double/double gets converted by (int) cast
  }
}

toString(), by default, returns the String version of the object you input (which itself is pretty useful), but you can override it to influence how System.out.println() works.

public class ToStringDemo{
  private int a;
  private int b;
  public ToStringDemo(int one, int two){
    a = one;
    b = two;
  }
  public static void main(String[] args){
    ToStringDemo thing = new ToStringDemo(2,3);
    System.out.println(thing); //printing the OBJECT triggers toString
  }
  //with this you can AUTOMATICALLY change how your objects get printed out.
  public String toString(){
    return a + "and" + b;
  }
}

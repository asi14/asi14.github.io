Program Theory

Good Workflow

There are four major steps to program development.

Requirements

Basic points your program must meet. They will be...

  • generic
  • stupid
  • vague

And might have little to do with your actual code!

Design

What your program basically look like?

  • Algorithims are sculpted here
  • Psuedocode (yes, that) can give a general sense of your code, without syntax

Don't dive into your code! Think it out first

This is what happens if you don't plan your code

Construction

Writing your code

  • It's good to go top-down- big stuff now, small stuff later
  • Assuming you know (generally) what you're doing, this should be straight-forward

Testing

Hoping your program doesn't crash

Errors can range from being structural to being caused by a forgotten ;

No matter how many errors you find, there's always the chance for more

No error is that bad- please don't break your computer

These steps aren't static- you'll be in various parts of these as you program.

Switch Blocks New

public class switchTest{
  public static void main(String[] args){
    int tester = 8;
    //basic stucture: switch(thing to test){case of test: run something}
    switch(tester){
      //in the possibility tester is 1
      case 1:
        System.out.println("number 1!");
        break; //DONT FORGET BREAK IN YOUR CASES OR ELSE YOUR CASES FALL THROUGH
      case 2: case 3: case 4: case 5: case 6: case 7: //you can stack cases together for same output
        System.out.println("Deez nuts");
      case 8:
        System.out.println("Fitness Gram Pacer Test");
    }
  }
}

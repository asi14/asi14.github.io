class Calc{
  //you know these as INSTANCE variables
  //they're instances of each object, just as each object is an instance of a class
  private int alpha;
  private int beta;

  //this is  STATIC variable: all objects share this same data
  //means if you change static variables for one object, it changes for all else
  private static int count;

  //here's a common usage of static variables: signifying "full" constants
  private static final int TYPE = 0;

  //constructors are always not static, cause they instantiate objects
  public Calc(int a, int b){
    alpha = a;
    beta = b;
  }
  public int add(){
    int sum = 0; //this is a LOCAL variable because it's called within a method
                 //this variable will die when the method ends
    count = 5;
    sum = a + b;
    return sum;
  }

  //this is a static method in implementation
  //it's called via the CLASS, so no objects!
  public static int subtract(){
    return alpha - beta; //this causes an error
    //static methods can't call instance variables
    //same goees vice versa
  }
}

Inheritance

Basic Inheritance

Child classes extends from Parent classes

Essentially, a child classe inherits all of its parent class's protected and public instance variables or methods. (private stuff isn't inherited... I wonder why)

Java uses extends to indicate inheritance. B extends A means A is B's parent class (and vice versa)

You can only do this ONCE by the way, meaning Java's single-inheritance and has is-a relationships.

Programmers cite reusability as inheritance's good points.

public class ClientClass{
  public static void main(String[] args){
    B obj = new B();
    B.printAnotherThing();
  }
}
class A{
  public void printSomething(){
    System.out.println("This is an A");
  }
}

class B extends A{
  public void printAnotherThing(){
    printSomething();
    System.out.println("But this is a B too!");
  }
}

Stack extends long enough and you'll get something called class heiarchies (this is what I mean)

super accesses a parent class's stuff

Two big usages of super is for calling parent constructors and methods.

public class ClientClass{
  public static void main(String[] args){
    ChildClass a = new ChildClass(3, "hello", 'p');
    a.thing();
  }
}

class ParentClass{
  protected int field1;
  protected String field2;
  public ParentClass(int a, String b){
    field1 = a;
    field2 = b;
  }
  public int increment(){
    return field1++;
  }
}

class ChildClass extends ParentClass{
  private char field3;
  public ChildClass(int a, String b, char c){
    super(a, b);
    field3 = c;
  }
  public void thing(){
    System.out.println(super.increment() + field3);
  }
}

Calling super() is a reference to your parent class's constructor.

By defeault all child classes make a super() (no paremeters) call. If you're extending you'll need to add a constructor in the parent class or you'll get an error.

super.method() is a reference to a parent class method

This can be used to access private parent variables made earlier with super (see the example).

Careful! To do this you'll need to make said parent variables public or protected.

Abstract Classes

The keyword abstract indicates a class contains abstract methods.

It means it has abstract method headers to be later implemented in child classes.

Abstract methods have the abstract keyword in the method header and no method body.

These methods are later implemented in child classes.

Warning: you can't instantiate abstract classes

NOTE: abstract is different from interface in that the latter is entirely headers

You can still have regualar methods in abstract

public abstract class AbstractClass{
  protected int a;
  public abstract int getSum(); //this must be implemented or it'll yield an error
  public void print(){
    System.out.println("I am a totmato");
  }
}
public class ChildClass extends AbstractClass{
  public int getSum(){
    return a++;
  }
  public void print(){
    super.print();
    System.out.println("I am a potato");
  }
}

public class ClientClass{
  public static void main(String[] args){
    ChildClass abba = new ChildClass();
    System.out.println(getSum() + print()); //what happens when print() is called?
  }
}
